## 기능 EPIC의 크기
- 현재 너무 자잘하게 나눠져서 EPIC이 Chain 수준으로 보임
- 좀 더 큰 단위로 나누는 것이 좋을 것 같음
- EPIC은 프로젝트 레벨
- Chain은 Feature 레벨

## EPIC 분할 후 검증
- MECE 원칙 준수
- 분할 후 체인의 PRD 문서와 EPIC의 PRD 문서를 누락이 있는지 일관성이 있는지 검토를 지속적으로 해야 
- 프롬프트 예 
```
@projects\jjiban\jjiban-prd.md 로 하위 업무 범위로 분할 했는데 MECE 원칙을 준수했는지 누락은 없는지 또 적절한 크기로 잘 나누었는지 검토 해줘.
```


## EPIC, Chain, Module, Task 분할 원칙
- 수동으로 나누는 것이 좋다. 특히 프로젝트가 클 경우, 미리 WBS를 생성한 후 Task 레벨로 분할
- 자동으로 나누는 것은 작은 프로젝트일 경우
- 자동으로 나누면 사용자가 원하는 크기로 나누어지지 않을 수 있으므로 수동으로 나누는 것이 좋음

##  AI 코딩 착수 프롬프트 

### 핵심 규칙

1. **스택 준수**: TRD.md에 명시된 라이브러리와 버전만 사용
2. **추측 금지**: 요구사항이 불명확하면 명확화 요청
3. **편법 금지**: 완전하고 프로덕션 레디 코드 구현
4. **아키텍처 우선**: 기능 코드 작성 전 프로젝트 구조 설정
5. **테스트 커버리지**: 구현과 함께 테스트 작성

### 실행 순서

1. TRD.md의 정확한 버전으로 프로젝트 초기화
2. 명시된 대로 디렉토리 구조 설정
3. 린팅, 포매팅, pre-commit 훅 설정
4. 우선순위 순서로 기능 구현 (P0 먼저)
5. 각 기능에 대한 테스트 작성
6. 모든 이탈 사항이나 결정 문서화
