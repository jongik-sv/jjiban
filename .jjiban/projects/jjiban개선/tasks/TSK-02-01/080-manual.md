**Version:** 1.0.0 — **Last Updated:** 2025-12-26

## 1. 개요

### 1.1 기능 소개

워크플로우 액션 UI는 jjiban 프로젝트 관리 도구에서 Task의 워크플로우를 제어하는 버튼 컴포넌트입니다.

**주요 기능**:
- 13개 워크플로우 명령어 버튼 제공
- Task 상태/카테고리에 따른 자동 활성화/비활성화
- Run, Auto, 중지 자동 실행 버튼

### 1.2 대상 사용자

- jjiban을 사용하는 개발자
- 프로젝트 관리자

---

## 2. 시작하기

### 2.1 접근 방법

WBS 페이지에서 Task를 선택하면 오른쪽 상세 패널에 워크플로우 액션 UI가 표시됩니다.

```
WBS 페이지 → Task 선택 → 상세 패널 → 워크플로우 액션
```

---

## 3. 사용 방법

### 3.1 명령어 버튼

| 명령어 | 라벨 | 설명 | 활성화 상태 |
|--------|------|------|------------|
| start | 시작 | 워크플로우 시작 | `[ ]` |
| ui | UI설계 | UI 설계 문서 생성 | `[bd]` |
| draft | 상세설계 | 상세설계 문서 생성 | `[bd]` |
| review | 리뷰 | 설계 리뷰 요청 | `[dd]` |
| apply | 적용 | 리뷰 결과 적용 | `[dd]` |
| build | 구현 | TDD 기반 구현 | `[dd]`, `[ds]` |
| test | 테스트 | 테스트 실행 | `[im]` |
| audit | 코드리뷰 | 코드 리뷰 요청 | `[im]`, `[fx]` |
| patch | 패치 | 리뷰 결과 패치 | `[im]`, `[fx]` |
| verify | 검증 | 통합테스트 | `[im]`, `[fx]` |
| done | 완료 | 작업 완료 | `[vf]`, `[im]` |
| fix | 수정 | 결함 수정 | `[an]` |
| skip | 생략 | 설계 생략 | `[ ]` (infrastructure) |

### 3.2 버튼 상태

| 상태 | 설명 | 스타일 |
|------|------|--------|
| 활성 | 클릭 가능 | 명령어별 색상 |
| 비활성 | 현재 상태에서 사용 불가 | 회색 (secondary) |
| 로딩 | 명령어 실행 중 | 스피너 표시 |

### 3.3 자동 실행 버튼

| 버튼 | 설명 |
|------|------|
| Run | 다음 단계 자동 실행 |
| Auto | 목표까지 자동 실행 |
| 중지 | 실행 중단 (실행 중에만 표시) |

## 4. FAQ

### Q1. 버튼이 모두 비활성화되어 있습니다.

현재 Task 상태에서 실행 가능한 명령어가 없거나, 이미 완료된 Task입니다.

### Q2. 버튼을 클릭해도 반응이 없습니다.

터미널 시스템과의 연동이 필요합니다. TSK-04-01 완료 후 사용 가능합니다.
