# PRD를 Epic으로 분리하는 범용 명령어

현재 프로젝트의 PRD 문서를 분석하여 Epic 단위로 분리하고, projects 폴더 구조를 생성합니다.

## ⚠️ 중요 지침

**이 명령어는 문서화 작업 전용입니다:**
- ✅ PRD 문서 분석 및 Epic 분할
- ✅ Epic PRD 문서 생성
- ✅ 폴더 구조 생성
- ✅ Feature 목록 자동 제안
- ✅ 의존성 분석 및 매핑
- ❌ **코드 작성 금지** - 구현은 별도 명령어 사용
- ❌ **코드 생성 금지** - 설계 문서만 작성

**비즈니스 요구사항 중심 분할 원칙:**
1. **기능적 가치 중심**: 기술 스택이 아닌 사용자/비즈니스 가치로 Epic 그룹핑
2. **도메인 중심**: 비즈니스 도메인과 기능 영역을 우선 고려
3. **독립적 가치 제공**: 각 Epic이 독립적으로 비즈니스 가치를 제공할 수 있도록
4. **사용자 시나리오 기반**: 사용자 여정과 워크플로우를 고려한 분할
5. **우선순위 반영**: 비즈니스 우선순위와 ROI를 고려한 Epic 구성

## 사전 요구사항

사용자에게 다음 정보를 요청하세요:
1. **PRD 파일 경로** (예: `./project-prd.md` 또는 `./docs/requirements.md`)
2. **Epic 분류 방식** (자동 추론 또는 수동 지정)
3. **프로젝트 루트 폴더** (기본값: `./projects/`)

## 실행 단계

### 1단계: PRD 문서 분석

1. PRD 파일 읽기
2. 문서 구조 파악 (섹션, 헤더 레벨)
3. 주요 기능 영역 식별:
   - 섹션 헤더 (## 또는 ###)
   - 키워드 패턴 (시스템, 기능, UI, 관리, 통합 등)
   - 논리적 그룹핑

### 2단계: Epic 추출

다음 기준으로 Epic을 자동 추출:

**기능 영역 키워드:**
- **관리**: "관리", "Management", "Admin"
- **UI/화면**: "UI", "화면", "View", "Board", "Chart"
- **통합**: "통합", "Integration", "연동", "API"
- **워크플로우**: "워크플로우", "Workflow", "프로세스"
- **시스템**: "시스템", "System", "엔진", "Engine"
- **배포**: "배포", "Deployment", "설치", "Installation"
- **문서**: "문서", "Document", "템플릿", "Template"

**Epic 추출 로직:**
```
1. 레벨 2 헤더(##)를 주요 섹션으로 파악
2. 각 섹션의 크기와 독립성 평가
3. 관련 섹션들을 하나의 Epic으로 그룹핑
4. Epic당 적절한 크기 유지 (너무 크거나 작지 않게)
5. 최소 3개, 최대 15개 Epic 추천
```

### 2-1단계: Epic 비즈니스 가치 평가

**⚠️ 중요: Epic은 비즈니스 가치를 제공하는 기능만 해당합니다.**

각 Epic 후보를 다음 기준으로 평가하세요:

**비즈니스 가치 평가 기준:**
1. ✅ **사용자 직접 가치**: 사용자가 직접 체감하는 기능인가?
2. ✅ **독립적 가치**: 다른 기능 없이도 단독으로 가치를 제공하는가?
3. ✅ **비즈니스 도메인**: 비즈니스 영역(인증, 프로젝트 관리 등)을 대표하는가?
4. ❌ **기술 레이어**: 단순히 기술적 구조(레이아웃, 컴포넌트)만 제공하는가?
5. ❌ **인프라**: 한 번 설정하고 끝나는 개발 인프라인가?

**Epic 분류:**

### A. 플랫폼 Epic
**정의:** 시스템이 작동하기 위해 반드시 필요한 모든 비기능 요구사항을 하나의 Epic으로 통합 관리

⚠️ **중요 원칙: 비기능 요구사항은 모두 하나의 Epic (EPIC-P01)으로 통합**
- 사용자가 명시하지 않았더라도 시스템 실행에 필수적인 모든 기능
- 여러 개의 플랫폼 Epic으로 분리하지 말고 **단일 Epic**으로 관리
- 세부 기능은 Feature로 분해

| Epic ID | Epic 이름 | 제공 가치 | 비고 |
|---------|----------|----------|------|
| **EPIC-P01** | 플랫폼 인프라 | 시스템 실행 필수 기능 | 사용자 관리, Portal, 보안, 로깅, DevOps 등 |

**선정 기준:**
- ✅ 사용자가 명시하지 않았지만 시스템이 작동하려면 반드시 필요
- ✅ 모든 기능 Epic의 기반이 되는 인프라
- ✅ 비기능 요구사항 (NFR: Non-Functional Requirements)
- ✅ 운영 안정성 및 보안을 위해 필수

**EPIC-P01에 포함되어야 하는 필수 시스템 기능 (비기능 요구사항):**


1. **Portal & 레이아웃 시스템**
   - 전역 헤더 (로고, 프로젝트 선택, 검색, 사용자 메뉴)
   - 사이드바 네비게이션
   - 레이아웃 템플릿 (기본, 전체화면, 분할)

2. **디자인 시스템 & 공통 컴포넌트**
   - 색상 팔레트, 타이포그래피
   - 공통 컴포넌트 (Button, Input, Modal, DataTable, Form, Tree 등)
   - UI 프레임워크 (Ant Design/Shadcn + Tailwind CSS)

3. **데이터베이스 스키마**
   - Prisma Schema 정의 (User, Epic, Chain, Module, Task)
   - 데이터베이스 초기화 및 마이그레이션

4. **사용자 관리 (User Management)**
   - User 테이블 스키마
   - 로그인/로그아웃 (JWT 인증)
   - 사용자 등록 및 프로필 관리
   - 역할 기반 접근 제어 (RBAC: Admin, ProjectManager, Developer, Viewer)
   - 비밀번호 찾기 및 재설정
   - Access Token (15분) + Refresh Token (7일)

5. **시스템 설정, 로깅, 에러 처리 & 보안**
   - **환경 변수 관리**: .env, dotenv
   - **시스템 설정**: config.json, llm-config.yaml
   - **로깅 시스템**: Winston (error, warn, info, debug)
   - **전역 에러 처리**: Express Error Handler, React Error Boundary
   - **보안 설정**:
     - CORS (Cross-Origin Resource Sharing)
     - Helmet (보안 헤더)
     - XSS 방지 (입력 Sanitization)
     - Rate Limiting (요청 제한)
     - SQL Injection 방지 (Prisma ORM)

6. **DevOps 및 배포 자동화**
   - Git 전략 및 브랜치 규칙
   - 커밋 메시지 규칙
   - CI/CD 파이프라인 (GitHub Actions)
   - npm 배포 설정

**⚠️ 중요:**
PRD를 Epic으로 분리할 때, 위의 필수 시스템 기능들을 자동으로 체크하고 빠진 항목이 있으면 EPIC-P01에 포함시켜야 합니다. 사용자가 명시하지 않았더라도 시스템이 안정적으로 작동하기 위해 반드시 필요한 기능입니다.

### B. 기능 Epic (EPIC-001~)
**정의:** 비즈니스 요구사항을 충족하는 핵심 기능

**예시:**
- EPIC-001: 대시보드 (프로젝트 현황 모니터링)
- EPIC-002: 프로젝트 관리 (Epic/Task 관리)
- EPIC-003: 칸반 보드 (시각적 워크플로우)
- EPIC-004: Gantt 차트 (일정 관리)

**선정 기준:**
- ✅ 사용자 시나리오 기반 (사용자가 "이것을 하려면...")
- ✅ 비즈니스 가치 측정 가능
- ✅ ROI 산정 가능

### C. 프로젝트 초기 설정 (docs/SETUP.md)
**정의:** 기술 인프라 및 개발 환경 설정 (Epic이 아님!)

**포함 항목:**
- ❌ Git 전략 및 브랜치 규칙
- ❌ CI/CD 파이프라인 (GitHub Actions)
- ❌ 데이터베이스 초기화 (Prisma Schema)
- ❌ 디자인 시스템 설정 (Ant Design/Shadcn)
- ❌ 시스템 설정 파일 (config.json, llm-config.yaml)
- ❌ 로깅 시스템 (Winston)
- ❌ 환경 변수 (.env)

**처리 방법:**
1. `docs/SETUP.md` 파일 생성
2. 모든 기술 설정을 이 문서에 통합
3. 각 기능 Epic에서 필요시 참조

**사용자 확인 절차:**

1. **플랫폼 Epic 확인**
   ```
   다음 플랫폼 Epic을 생성합니다:

   ✅ EPIC-P01: 플랫폼 인프라 (사용자 관리, Portal, 보안, 로깅, DevOps)
      → 모든 비기능 요구사항을 단일 Epic으로 통합

   진행하시겠습니까? (y/n)
   ```

2. **필수 시스템 기능 확인 (EPIC-P01)**
   ```
   EPIC-P01에 다음 필수 시스템 기능(비기능 요구사항)이 포함됩니다:

   ✅ Portal & 레이아웃 시스템
   ✅ 디자인 시스템 & 공통 컴포넌트
   ✅ 데이터베이스 스키마 (Prisma)
   ✅ 사용자 관리 (User 테이블, 로그인, JWT, RBAC)
   ✅ 환경 변수 관리 (.env)
   ✅ 시스템 설정 (config.json, llm-config.yaml)
   ✅ 로깅 시스템 (Winston)
   ✅ 전역 에러 처리 (Error Handler, Error Boundary)
   ✅ 보안 설정 (CORS, Helmet, XSS 방지, Rate Limiting)
   ✅ DevOps (Git 전략, CI/CD 파이프라인)

   이대로 진행하시겠습니까? (y/n)
   ```

**Epic ID 규칙:**
- 플랫폼 Epic: `EPIC-P01` (단일 Epic, 고정)
- 기능 Epic: `EPIC-001` ~ `EPIC-999`

예시:
```
플랫폼 Epic (비기능 요구사항 - 단일 Epic):
- EPIC-P01: 플랫폼 인프라 (사용자 관리, Portal, 보안, 로깅, DevOps)
  ├─ FEATURE-P01-001: Portal & 레이아웃
  ├─ FEATURE-P01-002: 디자인 시스템
  ├─ FEATURE-P01-003: 데이터베이스 스키마
  ├─ FEATURE-P01-004: 사용자 관리
  ├─ FEATURE-P01-005: 시스템 설정, 로깅, 에러 처리 & 보안
  └─ FEATURE-P01-006: CI/CD 파이프라인

기능 Epic (비즈니스 요구사항):
- EPIC-001: 대시보드 및 리포팅
- EPIC-002: 프로젝트 관리 시스템
- EPIC-003: 칸반 보드
- EPIC-004: Gantt 차트
```

**⚠️ EPIC-P01 자동 생성 규칙:**
PRD를 Epic으로 분리할 때, 다음 체크리스트를 확인하고 누락된 항목이 있으면 자동으로 EPIC-P01에 포함시켜야 합니다.
**모든 비기능 요구사항은 EPIC-P01 하나로 통합하고, 세부 기능은 Feature로 분해합니다.**

**필수 시스템 기능 체크리스트 (EPIC-P01):**
- [ ] Portal & 레이아웃 시스템
- [ ] 디자인 시스템 & 공통 컴포넌트
- [ ] 데이터베이스 스키마
- [ ] 사용자 관리 (User 테이블, 로그인, JWT, RBAC)
- [ ] 환경 변수 관리
- [ ] 시스템 설정 파일
- [ ] 로깅 시스템
- [ ] 전역 에러 처리
- [ ] 보안 설정
- [ ] DevOps (Git, CI/CD)

### 3단계: 폴더 구조 생성

```
{프로젝트 루트}/
├── docs/
│   └── SETUP.md                         # 프로젝트 초기 설정 가이드
│                                        # (Git, CI/CD, DB, 디자인 시스템 등)
│
└── projects/
    ├── EPIC-P01-platform/               # 플랫폼 Epic 폴더 (플랫폼 인프라)
    │   └── epic-prd.md
    │
    ├── EPIC-001-{epic-name}/            # 기능 Epic 폴더
    │   └── epic-prd.md
    ├── EPIC-002-{epic-name}/
    │   └── epic-prd.md
    ├── EPIC-003-{epic-name}/
    │   └── epic-prd.md
    └── ...
```

**Epic 네이밍 규칙:**
- **플랫폼 Epic ID**: `EPIC-P{2자리 숫자}` (예: EPIC-P01)
- **기능 Epic ID**: `EPIC-{3자리 숫자}` (예: EPIC-001, EPIC-002)
- **폴더명**: `EPIC-{ID}-{kebab-case-name}`
  - 플랫폼: `EPIC-P01-platform`
  - 기능: `EPIC-001-dashboard`, `EPIC-002-project-management`

### 5단계: 내용 분배 및 재구성

**분배 원칙:**
1. **중복 제거**: 공통 섹션은 요약하여 각 Epic에 포함
2. **컨텍스트 유지**: 각 Epic이 독립적으로 이해 가능하도록
3. **크로스 레퍼런스**: Epic 간 참조는 명시적으로 링크
4. **원본 유지**: 원본 PRD 경로를 명시하여 추적 가능

**자동 분배 로직:**
```
1. 섹션별 키워드 매칭으로 Epic 할당
2. 섹션이 여러 Epic과 관련되면 주 Epic에 할당, 나머지는 참조
3. 공통 섹션(개요, 목표 등)은 모든 Epic에 요약 포함
4. 기술 스택, 아키텍처는 관련 Epic에만 상세 포함
```

### 6단계: Feature 목록 자동 제안

각 Epic의 내용을 분석하여 3~7개의 Feature를 자동 제안:

**Feature 추출 기준:**
- 레벨 3 헤더(###)를 Feature 후보로 추출
- 독립적으로 구현 가능한 기능 단위
- 1~3개월 내 완료 가능한 크기
- 명확한 산출물이 있는 작업

**Feature 네이밍:**
```
FEATURE-{Epic ID}-{순번}: {명확한 기능 설명}

플랫폼 Epic 예시:
- FEATURE-P01-001: User 테이블 스키마 설계
- FEATURE-P01-002: 로그인/로그아웃
- FEATURE-P01-003: JWT 인증 및 토큰 관리

기능 Epic 예시:
- FEATURE-001-001: 대시보드 레이아웃
- FEATURE-001-002: 프로젝트 요약 카드
- FEATURE-002-001: Epic/Chain/Module 계층 구조
```

### 7단계: Epic 간 의존성 맵핑

**의존성 유형:**
1. **선행 의존성**: 이 Epic 시작 전에 완료되어야 할 Epic
2. **병렬 의존성**: 동시 진행 가능하지만 조율이 필요한 Epic
3. **후행 의존성**: 이 Epic 완료 후 시작할 수 있는 Epic

**의존성 표기:**
```markdown
### 3.3 의존성

**선행 Epic (Blocking Dependencies):**
- EPIC-001 (프로젝트 관리 시스템) - DB 스키마 필요
- EPIC-003 (문서 관리 시스템) - 문서 저장 구조 필요

**병렬 Epic (Parallel Dependencies):**
- EPIC-004 (칸반 보드 UI) - 공통 컴포넌트 공유
- EPIC-005 (Gantt 차트 UI) - 데이터 모델 공유

**후행 Epic (Dependent Epics):**
- EPIC-008 (워크플로우 자동화) - 이 Epic 완료 후 진행 가능
```

## 실행 지침

### 자동 모드 (추천)
```

```

## 품질 기준

- ✅ **독립성**: 각 Epic PRD는 독립적으로 이해 가능
- ✅ **완전성**: 원본 PRD의 모든 내용이 적절히 분배됨
- ✅ **일관성**: 모든 Epic이 동일한 템플릿 구조 사용
- ✅ **추적성**: 원본 PRD와의 연결 명확히 표시
- ✅ **실행 가능성**: Feature가 실제 구현 가능한 단위
- ✅ **명확한 의존성**: Epic 간 선후행 관계 명시

## 고급 옵션

사용자 요청 시 다음 옵션 지원:

1. **Epic 개수 제한**: `--max-epics N` (기본값: 15)
2. **최소 Feature 수**: `--min-features N` (기본값: 3)
3. **출력 디렉토리**: `--output-dir PATH` (기본값: ./projects/)
4. **템플릿 커스터마이징**: `--template PATH`
5. **의존성 시각화**: `--visualize-deps` (Mermaid 다이어그램 생성)
6. **WBS 자동 생성**: `--generate-wbs` (Excel/CSV 출력)
