# 넥사크로 N v24 주요 컴포넌트 목록

넥사크로 N v24는 웹 기반 RIA(Rich Internet Application) 개발을 위한 차세대 플랫폼으로, 다양한 내장 컴포넌트와 오브젝트를 제공합니다. 이 컴포넌트들은 넥사크로 스튜디오에서 시각적으로 배치하거나 스크립트를 통해 동적으로 생성하여 사용할 수 있습니다.

넥사크로 N v24의 주요 컴포넌트와 오브젝트는 기능에 따라 다음과 같이 분류할 수 있습니다.

---

## 1. 단일 컴포넌트 (Single Components)

사용자 인터페이스(UI)를 구성하는 가장 기본적인 요소들입니다.

### 기본 입력/출력 컴포넌트
- **Static**: 화면에 설명이나 제목 등 고정된 문자열을 출력합니다.
- **Button**: 사용자가 클릭하여 특정 동작(스크립트)을 실행시킬 수 있습니다.
- **Edit**: 한 줄로 된 짧은 문자열을 입력받거나 출력합니다.
- **MaskEdit**: 전화번호, 날짜 등 정해진 형식(Mask)에 따라 값을 입력받습니다.
- **TextArea**: 여러 줄로 된 긴 문자열을 입력받거나 출력합니다.
- **CheckBox**: 하나 또는 여러 개의 옵션을 선택하거나 해제할 때 사용합니다.
- **Spin**: 특정 범위 내의 숫자 값을 화살표로 증감시켜 입력받습니다.

### 날짜/시간 컴포넌트
- **Calendar**: 날짜 또는 시간을 쉽게 입력받을 수 있도록 달력을 제공합니다.
- **PopupCalendar**: 달력을 팝업 형태로 표시하는 컴포넌트입니다.
- **DatePicker**: 날짜 선택을 위한 편리한 인터페이스를 제공합니다.

### 미디어 및 시각화 컴포넌트
- **ImageViewer**: 이미지(jpg, png, gif, svg 등)를 화면에 표시합니다.
- **ProgressBar**: 작업의 진행 상태를 시각적인 바로 보여줍니다.
- **Sketch**: 사용자가 직접 서명을 하거나 그림을 그릴 수 있는 영역을 제공합니다.
- **VideoPlayer**: 동영상 파일을 재생합니다.
- **WebBrowser**: 폼 안에 작은 웹 브라우저를 삽입하여 외부 웹 페이지를 표시합니다.

### 파일 및 외부 연동 컴포넌트
- **FileDownload**: 서버에서 파일을 다운로드할 수 있는 기능을 제공합니다.
- **FileUpload**: 클라이언트에서 서버로 파일을 업로드할 수 있는 기능을 제공합니다.
- **Plugin**: 외부 플러그인을 화면에 삽입합니다.

### UI 확장 컴포넌트
- **PopupMenu**: 우클릭이나 특정 이벤트에서 나타나는 팝업 메뉴를 제공합니다.
- **InnerForm**: 폼 내부에 다른 폼을 삽입하여 화면을 구성할 수 있게 합니다.

---

## 2. 목록형 컴포넌트 (List Components)

여러 개의 아이템을 목록 형태로 보여주고 그중 하나 또는 여러 개를 선택할 수 있도록 합니다.

### 선택형 컴포넌트
- **Combo**: 드롭다운 목록에서 하나의 아이템을 선택합니다.
- **ListBox**: 목록 상자에서 하나 또는 여러 개의 아이템을 선택합니다.
- **Radio**: 여러 개의 항목 중 단 하나만 선택할 수 있도록 원형 버튼을 제공합니다.

### 고급 목록 컴포넌트
- **ListView**: 정형화되지 않은 자유로운 형태의 목록을 표현할 때 사용하며, 특히 모바일 환경에서 유용합니다.
- **Grid**: 데이터를 행과 열의 표 형태로 표시하고 편집할 수 있는 가장 핵심적인 컴포넌트입니다.
- **Tree**: 계층형 데이터를 트리 구조로 표시하고 확장/축소가 가능합니다.

### 프로세스 표시 컴포넌트
- **Step**: 단계별 진행 상황을 시각적으로 표시하는 컴포넌트입니다.

---

## 3. 컨테이너 컴포넌트 (Container Components)

다른 컴포넌트들을 담는 그릇 역할을 하며, 화면을 구조적으로 설계할 수 있도록 도와줍니다.

### 기본 컨테이너
- **Div**: 화면을 논리적인 단위로 분할하여 관리합니다.
- **PopupDiv**: 팝업 형태로 화면 위에 다른 컴포넌트들을 보여줄 때 사용합니다.
- **GroupBox**: 관련된 컴포넌트들을 시각적으로 그룹화하여 묶어줍니다.

### 탭 및 메뉴 컨테이너
- **Tab**: 여러 개의 페이지를 탭 형태로 구성하여 화면 공간을 효율적으로 사용할 수 있게 합니다.
- **Menu**: 계층형 메뉴 구조를 제공하여 다양한 메뉴 항목을 표시합니다.

### 분할 및 프레임 컨테이너
- **Splitter**: 화면을 여러 영역으로 분할하고 사용자가 경계선을 드래그하여 크기를 조절할 수 있게 합니다.
- **Form**: 다른 화면(폼)을 현재 화면에 삽입하여 화면 간 재사용성을 높입니다.
- **ChildFrame**: MDI(Multi Document Interface) 형태로 자식 화면을 관리합니다.
- **FrameSet**: 프레임셋을 이용하여 여러 화면을 분할하여 표시합니다.

---

## 4. 데이터 오브젝트 (Data Objects)

화면에 보이지 않으며, 데이터를 담거나 처리하는 역할을 합니다.

### 핵심 데이터 관리
- **Dataset**: 행(Row)과 열(Column)으로 구성된 2차원 테이블 형태의 데이터를 관리하는 핵심 오브젝트입니다. 서버와 통신할 때 데이터 교환의 기본 단위입니다.
- **DataObject**: 데이터셋과 달리, 서버와 데이터를 주고받을 때 좀 더 유연한 구조(JSON 등)로 통신할 수 있도록 지원합니다.
- **Variable**: 전역 변수나 애플리케이션 설정값을 저장하고 관리합니다.

### 통신 및 파일 처리
- **Transaction**: 서버와의 데이터 송수신을 담당하는 오브젝트로, HTTP 통신을 관리합니다.
- **ImageObject**: 이미지 데이터를 관리하고 처리하는 오브젝트입니다.

### 엑셀 연동
- **ExcelExportObject**: 데이터셋의 데이터를 엑셀 파일로 내보내는 기능을 제공합니다.
- **ExcelImportObject**: 엑셀 파일의 데이터를 읽어와 데이터셋으로 변환하는 기능을 제공합니다.

### 로컬 저장소
- **LocalStorage**: 클라이언트 로컬에 데이터를 저장하고 관리하는 기능을 제공합니다.
- **SessionStorage**: 세션 기반으로 데이터를 임시 저장하는 기능을 제공합니다.

## 5. 기타 오브젝트 (Etc. Objects)

애니메이션, 데이터 파싱, 시스템 연동 등 특수한 기능을 담당합니다.

### 애니메이션 및 시각 효과
- **Animation**: 컴포넌트에 움직이는 효과를 적용합니다.
- **AnimationTimeline**: 여러 애니메이션 효과를 시간 순서에 따라 제어합니다.

### 데이터 처리 및 파싱
- **nexacro.Decimal**: 소수점 연산을 정밀하게 처리합니다.
- **nexacro.JSON**: JSON 형식의 데이터를 파싱하고 처리합니다.
- **nexacro.XmlParser**: XML 형식의 데이터를 분석하고 처리합니다.

### 시스템 및 유틸리티
- **Timer**: 지정된 시간 간격으로 특정 작업을 반복 실행합니다.
- **VirtualFile**: 가상 파일 시스템을 통해 파일을 관리합니다.

### 출력 및 리포팅
- **PrintObject**: 프린터로 출력하는 기능을 제공합니다.
- **ReportObject**: 리포트 형태의 문서를 생성하고 출력하는 기능을 제공합니다.

### 실시간 통신
- **CommunicationObject**: 서버와의 실시간 통신을 위한 WebSocket 등의 기능을 제공합니다.

## 6. Nexacro N v24 신규 기능

### 향상된 웹 표준 지원
- **HTML5 호환성**: HTML5 표준을 준수하는 컴포넌트 렌더링
- **CSS3 지원**: 향상된 스타일링과 애니메이션 효과
- **반응형 디자인**: 다양한 화면 크기에 자동 적응

### 개발 생산성 향상
- **컴포넌트 재사용성**: 향상된 컴포넌트 모듈화 지원
- **테마 시스템**: 일관된 디자인 시스템 적용
- **디버깅 도구**: 강화된 개발자 도구와 디버깅 환경

### 성능 최적화
- **가상화 지원**: 대용량 데이터 처리를 위한 가상화 기술
- **메모리 관리**: 최적화된 메모리 사용량과 가비지 컬렉션
- **로딩 최적화**: 빠른 초기 로딩과 지연 로딩 지원

## 컴포넌트 사용 시 주의사항

1. **컨테이너 우선 생성**: 컨테이너 컴포넌트를 먼저 생성한 후 내부 컴포넌트를 배치해야 합니다.
2. **데이터 바인딩**: Dataset과 UI 컴포넌트 간의 데이터 바인딩을 통해 효율적인 데이터 관리가 가능합니다.
3. **이벤트 처리**: 각 컴포넌트별 고유한 이벤트를 활용하여 사용자 인터랙션을 처리합니다.
4. **성능 고려**: 대용량 데이터 처리 시 가상화나 페이징 기법을 활용하는 것이 권장됩니다.
5. **크로스 브라우징**: 다양한 브라우저 환경에서의 호환성을 고려하여 개발해야 합니다.
