## 📊 분석 요약

현재 JJIban 프로젝트는 **칸반 보드**, **웹 터미널**, **이슈 관리 API** 등 핵심 인프라는 구축되었으나, **문서 뷰어**, **프롬프트 템플릿 관리**, **워크플로우 자동화**, **Gantt 차트** 등 차별화 기능이 미구현 상태입니다.

## 🔴 1. 핵심 UI 컴포넌트 미구현

### 1.1 문서 뷰어 (Document Viewer)
**상태**: ❌ 완전 미구현  
**PRD 참조**: 섹션 4.3.4  
**중요도**: 🔥 CRITICAL

#### 필요 기능:
- ✅ Markdown 렌더링 (GitHub Flavored Markdown 지원)
  - `react-markdown` + `remark-gfm` 사용 권장
- ✅ 코드 하이라이팅
  - `rehype-highlight` 또는 `prism-react-renderer`
- ✅ Mermaid 다이어그램 렌더링
  - `mermaid.js` 통합
- ✅ PlantUML 지원 (선택적)
- ✅ 수식 렌더링 (KaTeX)
  - `remark-math` + `rehype-katex`
- ✅ 파일 네비게이션
  - 트리 뷰 (폴더 구조)
  - 최근 파일 목록
  - 검색 기능
- ✅ 실시간 미리보기
- ✅ 복사 버튼 (코드 블록)

#### 구현 위치:
```
/frontend/src/features/doc-viewer/
├── ui/

│   ├── FileNavigator.tsx
│   └── CodeBlock.tsx
├── hooks/
│   └── useDocumentContent.ts

```

### 2.2 컨텍스트 메뉴 시스템
**상태**: ⚠️ 부분 구현 (UI만 있고 동적 로딩 미구현)  
**PRD 참조**: 섹션 3.2, 6.3  
**중요도**: 🔥 CRITICAL

#### 현재 문제:
- 칸반 카드 우클릭 시 컨텍스트 메뉴는 표시되나
- **DB의 `visible_columns` 기반 필터링이 안 됨**
- 템플릿 선택 시 자동으로 터미널에 프롬프트가 주입되지 않음

#### 필요 개선:
- ✅ `/api/issues/:id/context-menu` 엔드포인트 호출
- ✅ 현재 이슈의 `status`에 맞는 템플릿만 표시
- ✅ 템플릿 선택 시 자동으로 변수 치환 후 터미널 실행

## 🔴 4. 워크플로우 자동화 (Auto-Pilot)

### 4.1 자동화 파이프라인 실행
**상태**: ⚠️ 백엔드 구조만 있고 실제 실행 로직 미구현  
**PRD 참조**: 섹션 4.6  
**중요도**: 🔥 CRITICAL

#### 필요 기능:
- ✅ **완전 자동 모드 (Fully Automated)**
  - 각 단계 완료 후 자동으로 다음 단계 진행
  - 에러 발생 시만 중단
- ✅ **반자동 모드 (Human-in-the-Loop)**
  - 각 단계 완료 후 승인 대기
  - 사용자가 결과 확인 후 승인 버튼 클릭
- ✅ 파이프라인 정의
  - 상세설계 → 설계리뷰 → 설계리뷰적용 → 구현 → 코드리뷰 → 코드리뷰적용 → 완료
- ✅ 단계별 상태 추적
  - pending, running, success, failed, waiting_approval
- ✅ 에러 처리 및 복구
  - 실패한 단계부터 재실행

#### 구현 위치:
- Backend: `WorkflowService.executeJob()` 완성 필요
- Frontend: `WorkflowPanel` 개선 필요
