## ⚠️ 발견된 이슈

### 🔴 P1 - 즉시 해결 필요
**[SCRIPT 편집 UI 보안·안전장치 부족]**
- **심각도**: Critical · **카테고리**: 보안/UX
- **설명**: SCRIPT 타입을 폼/에디터에서 직접 입력하도록 설계했으나, 허용 구문, 길이 제한, 미리보기/검증, 관리자 승인, 경고 메시지 등이 정의되지 않음.
- **영향**: 악성 스크립트 저장 → 실행 시 시스템 위험, 사용자 오입력으로 인한 서비스 중단.
- **개선안**: (1) SCRIPT 입력 전용 안전 에디터(문법 강조·Lint·Sandbox), (2) 길이/허용 함수 제한, (3) 저장 시 관리자 승인/리뷰, (4) 위험 경고 및 감사 로그 필수화.

### 🟠 P2 - 빠른 해결 권장
**[우선순위 드래그/정렬 UX & 동시성 가이드 부족]**
- 드래그, Up/Down 버튼 등 상호작용은 제시되지만 실시간 재배치 규칙, 저장 타이밍, 충돌 처리, Undo/Redo 정책이 미흡.

**[조건/결과 입력 Validation 세분화 필요]**
- 연산자에 따라 허용값(숫자, 범위, 리스트)·필수 필드·에러 메시지가 구체화되지 않음. → 동적 Validation/ToolTip/예시 추가 필요.

**[요청 중복/디바운스 정책 부족]**
- 저장/삭제/우선순위 변경 시 버튼 비활성, 진행 표시, in-flight 취소/재시도 전략이 없다.

### 🟡 P3 - 보통 우선순위
- **Dataset 스키마**: OP_1~20/RSLT_1~20, PRIORITY_ORDER, DESCRIPTION 등 필드 타입/길이/코드셋 정의 필요.
- **오류/빈 상태 UI**: ErrorCode 매핑, 다국어 메시지, 빈 상태 안내 컴포넌트 표준이 필요.
- **접근성**: 그리드 포커스, 드래그 대안(키보드), 모달 제어, 단축키 안내 등을 명시해야 함.

### 🟢 P4 - 개선 항목
- 변경 히스토리 탭/비교 뷰 제공 검토(이전 버전과 diff 보여주기).
- 활동 로깅: 규칙 생성/수정/삭제/우선순위 변경/스크립트 편집 로그 및 알림.

### 🔵 P5 - 참고 사항
- RL0300(검증/실행)과 연동할 수 있도록 규칙 테스트 실행 버튼/샌드박스 호출 인터페이스 확장 여지 고려.
