## ⚠️ 발견된 이슈

### 🔴 P1 - 즉시 해결 필요
**[SCRIPT 테스트 UI 안전장치 부족]**
- **심각도**: Critical · **카테고리**: 보안/UX
- **설명**: SCRIPT 타입 조건을 UI에서 직접 실행하지만, 허용 구문/길이/경고/승인 절차가 없다. 악성/오류 스크립트 입력 시 브라우저 정지 또는 Backend 위험.
- **개선안**: (1) SCRIPT 입력 길이·허용 함수 제한, (2) Web Worker 기반 미리보기/타임아웃, (3) 위험 경고 및 관리자 승인 옵션, (4) 실행 결과/오류 상세 로그 제공.

### 🟠 P2 - 빠른 해결 권장
**[결과 로그/리포트 UX 표준화 부족]**
- 성공/실패/경고 표시, 색상, 아이콘, 재시도 버튼 등 상태별 UX가 텍스트 수준. → Severity별 색상/아이콘 표준, 토스트 vs 다이얼로그 기준, 다운로드 기능 명시 필요.

**[요청 중복/디바운스 정책 미흡]**
- 테스트 실행 버튼 연타, 입력 변경 시 즉시 재실행 등 중복 요청 방지가 없다. → submitting 플래그, in-flight 취소, Queue 처리 정책 정의 필요.

**[입력 Validation 세분화]**
- 변수 타입별 입력 규칙(숫자/문자/Boolean), NULL 허용 여부, 다중 조건 입력 UI(AND/OR) 제한이 미흡.

### 🟡 P3 - 보통 우선순위
- **Dataset 스키마**: 실행 요청/결과(STATUS, OUTPUT, LOG, DURATION) 필드 타입/길이/널 허용 규칙 명시.
- **접근성**: 실행 결과 그리드/로그 뷰/히스토그램의 키보드 조작, 스크린리더 라벨 강화 필요.
- **보안**: 테스트 입력 값/로그를 마스킹하거나 민감 정보 필터링 지침 추가 필요.

### 🟢 P4 - 개선 항목
- 실행 이력 탭/검색/필터 기능 확장 검토.
- 실행 시간·성공률 등 KPI를 차트로 시각화하여 운영 가시성 강화.

### 🔵 P5 - 참고 사항
- 향후 자동 회귀 테스트/CI 연동을 고려해 Test Suite 저장·재실행 API와 UI 확장 계획 수립 권장.
