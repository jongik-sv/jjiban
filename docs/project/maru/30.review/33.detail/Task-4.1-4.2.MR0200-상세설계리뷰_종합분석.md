## 🎯 프로젝트 맥락 요약

### 시스템 특성
- **단계**: PoC (Proof of Concept)
- **아키텍처**: 3-Tier (Nexacro N ↔ Node.js + Express ↔ SQLite → Oracle)
- **사용자**: 단일 관리자 (동시성 최소)
- **목표**: 레거시 GLUE 프레임워크 → 신규 시스템 전환

### 기술 스택
- **Frontend**: Nexacro N V24 (XML Dataset, Desktop 중심)
- **Backend**: Node.js v20.x, Express 5.x, knex.js
- **Database**: SQLite 3 (PoC) → Oracle (Production)
- **Cache**: node-cache (PoC) → Redis (향후)

### PoC 제약사항
- ✅ **포함**: 핵심 CRUD, 검색, 통계, 기본 캐싱
- ❌ **제외**: 인증/인가, 고급 보안, 복잡한 동시성 제어

### 플랫폼 제약
- **Nexacro XML 형식 필수**: JSON 미지원
- **HTTP 200 + ErrorCode 방식**: Nexacro 호환성
- **숫자 → 문자열 변환**: Nexacro Dataset 제약
- **UI 자동화 테스트 제한**: Playwright 제한적
