### 🟠 P2 - 빠른 해결 권장
**[이슈-01] 캐시 무효화 전략의 비효율성 가능성**
- **심각도**: Medium
- **카테고리**: 성능
- **설명**: 버전 변경 시 `maru:{maruId}:*` 패턴으로, 상태 변경 시 `list:maru:*` 패턴으로 캐시를 삭제하도록 설계되었습니다. 이는 관련 없는 캐시까지 모두 삭제하여 캐시 효율을 떨어뜨리고, 특정 상황에서 DB 부하를 가중시킬 수 있습니다.
- **영향**: 캐시 히트율이 저하되고, 데이터 변경이 잦은 마루의 경우 DB 조회 비용이 예상보다 증가할 수 있습니다.
- **개선안**: 더 세분화된 캐시 키 전략을 도입하고, 무효화 시 직접적인 관련이 있는 키만 정확히 삭제하도록 개선을 권장합니다. 예를 들어, 목록 캐시는 페이지 번호, 필터 조건까지 키에 포함시켜 다른 조건의 목록 캐시는 유지되도록 설계합니다.
- **참조**: 상세설계서 §5.3.2, §11.2.2

### 🟡 P3 - 보통 우선순위
**[이슈-02] 데이터 접근 계층(Repository)의 부재**
- **심각도**: Medium
- **카테고리**: 아키텍처
- **설명**: 상세설계서에는 `HistoryService`가 `knex.js`를 직접 사용하여 DB에 접근하는 것으로 보입니다. 이는 비즈니스 로직과 데이터 접근 로직의 결합도를 높여 유지보수성을 저해할 수 있습니다.
- **영향**: 향후 DB 변경(예: Oracle → PostgreSQL)이나 ORM 교체 시 서비스 계층의 코드 수정이 불가피해집니다. 또한, 테스트 시 DB 의존성이 높아져 단위 테스트가 복잡해집니다.
- **개선안**: 명시적인 Repository 패턴을 도입하여 데이터베이스 접근 로직을 서비스 계층과 완전히 분리할 것을 권장합니다. `HistoryRepository`와 같은 추상화된 계층을 추가하여 서비스는 비즈니스 로직에만 집중하도록 구조를 개선해야 합니다.
- **참조**: 상세설계서 §4, §5.1

**[이슈-03] 민감 정보 로깅 정책의 구체성 부족**
- **심각도**: Low
- **카테고리**: 보안
- **설명**: 로그 마스킹 정책이 언급되었으나, 어떤 정보(예: 사용자 개인정보, 내부 시스템 경로 등)를 민감 정보로 분류하고 어떻게 마스킹할지에 대한 구체적인 규칙이 정의되지 않았습니다.
- **영향**: 개발자마다 마스킹 기준이 달라져 중요 정보가 로그에 노출될 수 있는 잠재적 보안 위험이 존재합니다.
- **개선안**: 프로젝트 차원에서 민감 정보 목록을 정의하고, `password`, `token`, `user_name` 등 특정 키워드에 대한 자동 마스킹 규칙을 로깅 서비스에 명시적으로 설계해야 합니다.
- **참조**: 상세설계서 §10.1.2

### 🟢 P4 - 개선 항목
**[이슈-04] 모니터링 지표 수집 도구 명시 필요**
- **심각도**: Low
- **카테고리**: 품질
- **설명**: P50, P95, P99 응답 시간, 캐시 히트율 등 모니터링 지표는 잘 정의되었으나, 이를 수집하고 시각화할 도구(예: Prometheus, Grafana, Datadog)에 대한 언급이 없습니다.
- **영향**: 개발 단계에서 성능을 지속적으로 추적하고 병목 지점을 식별하는 데 어려움을 겪을 수 있습니다.
- **개선안**: PoC 단계이더라도 간단한 오픈소스 도구(예: `express-status-monitor`)를 도입하거나, 향후 확장할 모니터링 스택을 명시하여 성능 관리 기반을 마련할 것을 권장합니다.
- **참조**: 상세설계서 §5.4.1, §11.1

## 🎯 종합 평가
- **전반적인 설계 품질**: 매우 우수. 공통 서비스의 핵심 기능이 체계적으로 설계되었으며, 테스트와 확장성까지 고려한 성숙한 설계 문서입니다.
- **구현 준비도**: 높음. 발견된 이슈들은 구현에 치명적인 결함이라기보다, 시스템의 장기적인 유지보수성과 성능을 향상시키기 위한 개선 항목에 가깝습니다. P2, P3 이슈 해결 후 즉시 구현 착수가 가능합니다.
- **주요 리스크**: 캐시 무효화 전략이 개선되지 않을 경우, 데이터 변경이 잦은 운영 환경에서 성능 저하의 원인이 될 수 있습니다.
