### 🟠 P2 - 빠른 해결 권장

**API 엔드포인트 불일치**
- **심각도**: High
- **카테고리**: 아키텍처
- **설명**: 기본설계서에는 `POST /api/v1/maru-headers/{마루ID}/validate-codes`로 명시된 반면, 상세설계서에서는 `GET /api/v1/codes/validate`로 명시됨.
- **영향**: 프론트엔드-백엔드 연동 시 문제 발생할 수 있음
- **개선안**: API 명세서와 기본설계서, 상세설계서 간 엔드포인트를 일치시키고 공통된 API 문서를 유지해야 함
- **참조**: API 설계서 (3. api-design.md)

**성능 요구사항 불일치**
- **심각도**: Medium
- **카테고리**: 성능
- **설명**: 상세설계서의 성능 요구사항(10만 건 기준 10초 이내 응답)이 기본설계서의 기대치(100개 코드 검증: 1초 이내)와 차이남.
- **영향**: 시스템 성능 목표가 불명확함
- **개선안**: 일관된 성능 요구사항 정의 필요. 10만 건 대량 처리 vs 일반 조회 성능 요구사항을 구분하여 명시해야 함
- **참조**: business-requirements.md, 상세설계서 성능 섹션
