## ⚠️ 발견된 이슈

### 🔴 P1 - 즉시 해결 필요
**[중복 요청/경합에 대한 UI 제어 부재]**
- **심각도**: High
- **카테고리**: 품질/아키텍처
- **설명**: 검색/페이징/통계 재조회 등 다중 액션에서 in-flight 요청 취소, 버튼 비활성, 스로틀/디바운스 정책이 문서화되지 않음.
- **영향**: 불필요한 서버 부하, 결과 플리커, 페이지 번호 역전 등 UX 저하.
- **개선안**: 공통 전송 상태 플래그, 300ms 디바운스, 동일 파라미터 중복요청 병합, 취소 토큰 적용.

### 🟠 P2 - 빠른 해결 권장
**[페이징·정렬 계약 표준화]**
- **심각도**: Medium
- **카테고리**: 아키텍처/품질
- **설명**: page/limit/sort/order 파라미터 명세와 에러 처리(범위 초과, 음수) 기준이 산발적.
- **개선안**: 공통 파라미터 규약과 기본값 정의, UI 제약(최대 페이지, 허용 limit 세트).

**[통계 카드 캐시/동기화]**
- **심각도**: Medium
- **카테고리**: 성능/품질
- **설명**: 목록/통계 동기화와 캐시 무효화 트리거(clear-cache) 타이밍 정의가 부족.
- **개선안**: 목록 성공 시 통계 재조회 조건, clear-cache 후 강제 갱신, TTL 기반 재조회 정책.

**[에러/로딩 UX 가이드 보강]**
- **심각도**: Medium
- **카테고리**: 품질/UX
- **설명**: 글로벌 로딩 인디케이터, 그리드 빈상태/오류 표시, 재시도/취소 흐름 미흡.
- **개선안**: 상태별 UI(loading/empty/error) 컴포넌트/표준 메시지 정의.

### 🟡 P3 - 보통 우선순위
**[Dataset 스키마 명세 상세화]**
- 필드 타입/길이/널 허용/기본값/코드셋 표로 정리, JSON↔Dataset 매핑 테이블 포함.
