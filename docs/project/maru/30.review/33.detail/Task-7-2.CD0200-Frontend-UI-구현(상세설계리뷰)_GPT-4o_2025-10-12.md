## ⚠️ 발견된 이슈

### 🔴 P1 - 즉시 해결 필요
**[별칭(ALTER_CODE_NAME) 중복/충돌 실시간 검증 미비]**
- **심각도**: High · **카테고리**: 품질/UX/데이터 무결성
- **설명**: UI-REQ-006에 언급된 별칭 입력(최대 5개)에 대해 문서상 중복 검사/자동 추천/충돌 경고 기준이 없다. 동일 별칭이 다른 코드에 존재하거나 현재 코드의 다른 별칭과 중복될 때의 UX가 정의되지 않음.
- **영향**: 저장 시 백엔드 오류 발생, 사용자 혼란, 중복 데이터 유입.
- **개선안**: (1) 입력 단계에서 실시간 중복 조회(API or 로컬 캐시) 및 경고/포커스 이동, (2) 대소문자/공백 정규화 정책 명시, (3) 충돌 해결 UX(자동 번호 부여, 덮어쓰기 확인) 정의.

### 🟠 P2 - 빠른 해결 권장
**[정렬 재배치 UX & 동시성 정책 부족]**
- Drag&drop 또는 Up/Down 버튼 등 정렬 방식이 추상적으로만 언급. 순서 변경 시 배치 저장/실시간 반영 기준, 다중 선택 정책, 전송 실패 롤백 UX 명시 필요.

**[검색/필터 입력 디바운스 및 요청 병합 기준 미흡]**
- 검색어/별칭/상태 필터 변경 시 즉시 재조회한다고만 되어 있음. 300ms 디바운스, in-flight 취소, 버튼 확정 등 정책을 정하여 UX와 성능을 보장 필요.

**[오류/빈 상태/로딩 컴포넌트 표준 부재]**
- 에러 메시지 문구는 나열되었으나, 그리드/폼의 상태별(loading/empty/error) 레이아웃, 재시도 버튼, 포커스 이동 기준이 구체화되지 않음.

### 🟡 P3 - 보통 우선순위
- **Dataset 스키마**: ALTER_CODE_NAME1~5, SORT_ORDER, STATUS 등 필드의 길이/허용 문자/널 정책을 표로 정리하고 JSON↔Dataset 매핑 명시 필요.
- **접근성**: Grid 셀 aria-label, 모달 접근 통제, 단축키 안내, Tab/Shift+Tab 순서 등 세부 기준 강화 필요.

### 🟢 P4 - 개선 항목
- 활동 로깅: 별칭 편집/정렬 변경/삭제 시 사용자·시간·변경 전후 값을 기록하여 추후 감사 및 UX 개선 지표 확보.
- 국제화: 메시지/레이블 리소스를 별도 파일로 분리하여 번역 변경 시 재배포 최소화.

### 🔵 P5 - 참고 사항
