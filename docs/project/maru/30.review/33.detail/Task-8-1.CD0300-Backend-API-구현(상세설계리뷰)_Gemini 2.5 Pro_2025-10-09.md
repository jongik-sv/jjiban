### 🟠 P2 - 빠른 해결 권장
**[이슈 제목] API 엔드포인트 설계의 비일관성**
- **심각도**: High
- **카테고리**: 아키텍처
- **설명**: 상세설계서(`Task-8-1.CD0300...`)에는 API 엔드포인트가 `POST /api/v1/maru-headers/{maruId}/validate-codes`로 정의되어 있습니다. 그러나 `program-list.md`의 API 목록에는 `CB006` API가 `POST /api/v1/maru-headers/{id}/validate-codes`로 명시되어 있어 리소스 식별자(`id` vs `maruId`)와 경로 세분화(`validate-codes`)에서 차이가 있습니다. 또한, 다른 API들은 `codeCategory.js`, `codeBase.js` 등으로 파일이 분리되는데 반해, 이 API는 어느 라우터에 속할지 명확하지 않습니다.
- **영향**: API의 일관성을 해치고, Frontend 개발자가 API를 호출할 때 혼란을 유발할 수 있습니다. 향후 API 관리 및 유지보수를 어렵게 만듭니다.
- **개선안**:
  1. API 엔드포인트를 `POST /api/v1/maru-headers/{maruId}/codes/validation` 또는 `POST /api/v1/validations/codes/{maruId}` 와 같이 다른 RESTful API와 일관된 패턴으로 수정하는 것을 권장합니다.
  2. `program-list.md`와 상세설계서의 API 엔드포인트 정의를 일치시키고, 해당 API를 담당할 라우터 파일(예: `codeValidation.js`)을 명확히 정의해야 합니다.
- **참조**: `docs/project/maru/00.foundation/02.design-baseline/program-list.md` (API ID: CB006)

### 🟡 P3 - 보통 우선순위
**[이슈 제목] 대량 데이터 검증 시 성능 저하 가능성**
- **심각도**: Medium
- **카테고리**: 성능
- **설명**: 상세설계서 11.1절에 1000개 코드 검증 시 5초 이내 응답을 목표로 하고 있으나, 모든 코드를 DB에서 조회한 후 애플리케이션 메모리에서 반복적으로 검증하는 로직은 데이터가 수만 건 이상으로 증가할 경우 심각한 성능 저하를 유발할 수 있습니다.
- **영향**: 대량의 코드를 가진 마루(Maru) 검증 시, API 타임아웃이 발생하거나 서버에 과도한 부하를 줄 수 있습니다.
- **개선안**:
  1. **비동기 처리 도입**: 요청 시 즉시 작업 ID를 반환하고, 별도 API를 통해 진행 상태 및 결과를 조회하는 비동기 방식으로 전환하는 것을 장기적으로 고려해야 합니다. (설계서에 언급은 되어 있으나, 구체적인 설계 필요)
  2. **DB 레벨 검증**: 가능하면 복잡한 정규식을 제외한 `Normal` 타입 검증이나 중복 검증은 DB 쿼리 레벨에서 처리하여 애플리케이션 부하를 줄이는 방안을 검토해야 합니다.
- **참조**: `docs/project/maru/10.design/12.detail-design/Task-8-1.CD0300-Backend-API-구현(상세설계).md` (섹션 11)

### 🟢 P4 - 개선 항목
**[이슈 제목] Nexacro XML 응답 생성 로직의 위치**
- **심각도**: Low
- **카테고리**: 품질
- **설명**: `api-design.md` 문서에 `createNexacroResponse` 헬퍼 함수 예제가 포함되어 있습니다. 이 로직은 모든 컨트롤러에서 반복적으로 사용될 가능성이 높으므로, 공통 유틸리티 또는 미들웨어로 분리하여 관리하는 것이 좋습니다.
- **영향**: 코드 중복이 발생하고, 향후 XML 응답 구조 변경 시 여러 파일을 수정해야 하는 번거로움이 있습니다.
- **개선안**: `backend/src/utils` 디렉터리에 `nexacroResponse.js`와 같은 공통 모듈을 생성하고, 이를 컨트롤러에서 `import`하여 사용하도록 아키텍처 가이드에 명시하는 것을 권장합니다.
- **참조**: `docs/project/maru/00.foundation/02.design-baseline/api-design.md` (섹션 10.1)
