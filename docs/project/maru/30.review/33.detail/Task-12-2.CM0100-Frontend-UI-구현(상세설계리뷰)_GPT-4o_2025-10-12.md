## ⚠️ 발견된 이슈

### 🔴 P1 - 즉시 해결 필요
**[민감 정보 노출 방지·권한 제어 미비]**
- **심각도**: Critical · **카테고리**: 보안/UX
- **설명**: 시스템 부하, 에러 수, 최근 활동 로그 등 민감 정보가 표시되지만, 로그인/권한(RBAC)·마스킹·뷰 정책이 UI 문서에 없다.
- **영향**: 권한 없는 사용자가 운영 정보를 열람 가능 → 보안 사고/컴플라이언스 위반.
- **개선안**: (1) ROLE별 위젯 가시성 정의, (2) 민감 지표 마스킹/범주화, (3) 세션 만료/자동 로그아웃, (4) 감사 로그/알림 연계 명시.

### 🟠 P2 - 빠른 해결 권장
**[실시간 새로고침/자동 갱신 정책 세분화 부족]**
- 30초 타이머만 언급. 에러 시 재시도, 백오프, 사용자 수동 새로고침, 진행 표시 등 UX가 필요.

**[빈 상태/오류 상태/로딩 UX 표준화 미흡]**
- 카드별 상태 메시지, 색상, 아이콘, 재시도 버튼 규칙이 없다.

**[성능/렌더링 전략]**
- 복수 차트/카드 동시 렌더링에 대한 스켈레톤, 가상화, 해상도/반응형 가이드가 부족.

### 🟡 P3 - 보통 우선순위
- **Dataset 스키마**: 각 위젯 데이터(단위, 소수점, 타임존) 명시 필요.
- **접근성**: 키보드 내비게이션, 스크린리더 레이블, 대비 기준을 정리해야 함.
- **알림/바로가기**: 클릭 로그, 권한 검증, 외부 링크 보안 정책(새 창, HTTPS) 명시 필요.

### 🟢 P4 - 개선 항목
- 사용자 커스터마이징(위젯 숨김/정렬) 및 즐겨찾기 저장 기능 검토.
- 대시보드 상태를 CSV/PDF 등으로 내보내는 기능 확장 고려.

### 🔵 P5 - 참고 사항
- Observability 플랫폼(Grafana 등) 연계 시 UI 범위를 조정하고, 링크 통합 전략을 준비하면 확장성 향상.
