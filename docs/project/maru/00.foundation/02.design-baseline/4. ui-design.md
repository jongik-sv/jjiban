# MARU UI 기본설계서

**프로젝트**: MARU (Master Code & Rule Management System)
**문서 버전**: 1.0
**작성일**: 2025-09-18
**문서 유형**: 기본설계서

---

## 1. 개요

### 1.1 목적
MARU 시스템의 사용자 인터페이스 설계 및 Nexacro N v24 기반 화면 구성을 정의한다.

### 1.2 범위
- UI 설계 원칙
- 화면 구성 및 레이아웃
- 컴포넌트 설계
- 사용자 경험 (UX) 가이드

---

## 2. UI 설계 원칙

### 2.1 사용성 원칙
- **직관성**: 최소한의 학습으로 사용 가능
- **일관성**: 모든 화면에서 동일한 패턴 적용
- **효율성**: 최소 클릭으로 목표 달성
- **접근성**: 다양한 사용자 환경 고려

### 2.2 Nexacro N v24 설계 가이드
- **Form 기반 설계**: 기능별 독립적인 Form
- **Dataset 활용**: 효율적인 데이터 바인딩
- **Component 재사용**: 공통 컴포넌트 활용
- **반응형 레이아웃**: 다양한 해상도 지원
- **향상된 성능**: v24의 개선된 렌더링 엔진 활용
- **모던 웹 표준**: ES6+ 지원 및 웹 접근성 개선

```

│                    Title Bar                            │

│ Menu │                                                  │

│      │                                                  │

│                   Status Bar                            │
└─────────────────────────────────────────────────────────┘
```

### 3.2 영역별 구성요소

#### 3.2.1 Title Bar
- 시스템명: "MARU - 마스터 코드 및 룰 관리"
- 사용자 정보 (PoC에서는 고정)

- 계층형 메뉴 구조

#### 3.2.3 Content Area
- 선택된 메뉴에 따른 Form 표시
- 탭 구조로 다중 화면 지원
- 모달 팝업 지원

#### 3.2.4 Status Bar
- 현재 작업 상태
- 메시지 표시 영역
- 진행률 표시

## 5. 주요 화면 설계

### 5.1 마루 헤더 관리 (MR0100)

#### 5.1.1 화면 구성
```
┌─────────────────────────────────────────────────────────┐
│ [검색조건]                                               │
│ 마루타입: [CODE▼] 상태: [ALL▼] 검색어: [______] [검색]     │
├─────────────────────────────────────────────────────────┤
│ [신규] [수정] [삭제] [상태변경]                  총 50건  │
├─────────────────────────────────────────────────────────┤
│ □ │마루ID      │마루명        │타입│상태    │버전│시작일  │
│ □ │DEPT_001    │부서코드      │CODE│INUSE   │ 2 │2025-01-01│
│ □ │RULE_001    │급여계산룰    │RULE│CREATED │ 1 │2025-01-15│
│ ...                                                      │
├─────────────────────────────────────────────────────────┤
│ [상세정보]                                               │
│ 마루ID: DEPT_001           마루명: 부서코드               │
│ 타입: CODE                상태: INUSE                    │
│ 우선순위사용: N            버전: 2                        │
│ 시작일: 2025-01-01        종료일: 9999-12-31              │
└─────────────────────────────────────────────────────────┘
```

#### 5.1.2 주요 기능
- **조회**: 조건별 마루 목록 조회
- **신규**: 새로운 마루 생성
- **수정**: 선택된 마루 정보 수정
- **삭제**: 논리적 삭제 (종료일 갱신)
- **상태변경**: CREATED → INUSE → DEPRECATED

#### 5.1.3 Dataset 구조
```javascript
// ds_maruList: 마루 목록
{
  columns: [
    "MARU_ID", "VERSION", "MARU_NAME", "MARU_STATUS",
    "MARU_TYPE", "PRIORITY_USE_YN", "START_DATE", "END_DATE"
  ]
}

// ds_maruDetail: 마루 상세
{
  columns: [
    "MARU_ID", "MARU_NAME", "MARU_TYPE", "MARU_STATUS",
    "PRIORITY_USE_YN", "VERSION", "START_DATE", "END_DATE"
  ]
}
```

### 5.2 코드 기본값 관리 (CD0200)

#### 5.2.1 화면 구성
```
┌─────────────────────────────────────────────────────────┐
│ 마루선택: [DEPT_001 - 부서코드▼]                         │
├─────────────────────────────────────────────────────────┤
│ [신규] [수정] [삭제] [저장] [취소]                        │
├─────────────────────────────────────────────────────────┤
│ 코드      │코드명        │순번│대체코드명1     │대체코드명2 │
│ DEPT001   │경영지원팀    │ 1 │Management      │経営支援    │
│ DEPT002   │개발팀        │ 2 │Development     │開発       │
│ DEPT003   │영업팀        │ 3 │Sales          │営業       │
│ ...                                                      │
├─────────────────────────────────────────────────────────┤
│ [코드 검증]                                              │
│ 코드 정의: 1P,82,83,84,85,91,92                          │
│ 검증 결과: ✓ 모든 코드가 정의와 일치합니다                │
└─────────────────────────────────────────────────────────┘
```

#### 5.2.2 주요 기능
- **마루 선택**: 드롭다운으로 대상 마루 선택
- **코드 관리**: 인라인 편집으로 코드값 관리
- **정렬**: 드래그 앤 드롭으로 순번 변경
- **검증**: 카테고리 정의와 일치성 확인
- **다국어**: 대체 코드명 5개까지 지원

### 5.3 룰 레코드 관리 (RL0200)

#### 5.3.1 화면 구성
```
┌─────────────────────────────────────────────────────────┐
│ 마루선택: [RULE_001 - 급여계산룰▼]                        │
├─────────────────────────────────────────────────────────┤
│ [신규] [수정] [삭제] [복사] [테스트]                       │
├─────────────────────────────────────────────────────────┤
│ 우선순위│설명          │조건1     │조건2     │결과1     │  │
│    1    │임원급여      │등급=임원 │연차>=10  │급여*2.5  │  │
│    2    │관리자급여    │등급=관리 │연차>=5   │급여*2.0  │  │
│    3    │일반직급여    │등급=일반 │         │급여*1.5  │  │
│ ...                                                      │
├─────────────────────────────────────────────────────────┤
│ [룰 상세 편집]                                           │
│ 우선순위: [1    ] 설명: [임원 급여 계산 룰]               │
│ ┌─ 조건 ──────────────┐ ┌─ 결과 ──────────────┐       │
│ │OP1: [=] [임원] [   ]│ │RSLT1: [급여 * 2.5]  │       │
│ │OP2: [>=] [10] [   ] │ │RSLT2: [Y]          │       │
│ └─────────────────────┘ └─────────────────────┘       │
└─────────────────────────────────────────────────────────┘
```

#### 5.3.2 주요 기능
- **동적 컬럼**: 룰 변수 정의에 따라 컬럼 생성
- **매트릭스 편집**: 조건/결과 매트릭스 형태 편집
- **우선순위**: 드래그 앤 드롭으로 순서 변경
- **룰 테스트**: 입력값으로 룰 실행 테스트
- **조건 연산자**: =, !=, <, >, <=, >=, BETWEEN, IN 등

  properties: ["maruType", "status", "allowEmpty"],
  events: ["onMaruSelected"],
  dataset: "ds_maruList"

// comp_codeCombo

  properties: ["maruId", "categoryId", "allowEmpty"],

// comp_dateRange

## 7. 화면 전환 및 네비게이션

### 7.1 화면 전환 방식
- **메뉴 클릭**: 새로운 탭으로 열기
- **팝업**: 상세 편집이나 선택용
- **드릴다운**: 마스터-디테일 관계

### 7.2 데이터 플로우
```
Form Load → Dataset Initialize → Server Request → Data Binding → User Interaction
```

### 7.3 상태 관리
- **Form 상태**: 조회, 편집, 저장 등
- **Dataset 상태**: 원본, 수정, 신규 등
- **UI 상태**: 활성화, 비활성화, 로딩 등

---

## 8. 사용자 경험 (UX) 가이드

### 8.1 피드백 제공
- **진행 상태**: 로딩 인디케이터
- **성공/실패**: 토스트 메시지
- **검증 오류**: 인라인 에러 메시지
- **확인 요청**: 모달 다이얼로그

### 8.2 키보드 지원
- **Tab 순서**: 논리적 순서로 설정
- **단축키**: F2(편집), F5(새로고침), Ctrl+S(저장)
- **Enter**: 기본 액션 수행
- **ESC**: 취소 또는 닫기

### 8.3 접근성 고려사항
- **색상**: 색상에만 의존하지 않는 정보 전달
- **폰트**: 최소 12px 이상 크기
- **대비**: 충분한 색상 대비 (4.5:1 이상)
- **포커스**: 명확한 포커스 표시

---

## 9. 반응형 설계

### 9.1 해상도 지원
- **최소 해상도**: 1024 x 768
- **권장 해상도**: 1920 x 1080
- **최대 해상도**: 제한 없음

### 9.2 레이아웃 조정
- **그리드**: 컬럼 너비 자동 조정
- **폼**: 비율 기반 레이아웃
- **텍스트**: 상대적 크기 사용

---

## 10. 성능 최적화

### 10.1 데이터 로딩
- **페이징**: 기본 20건씩 조회
- **가상 스크롤**: 대량 데이터 그리드
- **캐시**: 자주 사용되는 데이터 캐시
- **지연 로딩**: 필요시에만 데이터 로드

### 10.2 렌더링 최적화
- **Double Buffering**: 화면 깜빡임 방지
- **컴포넌트 재사용**: 동일 컴포넌트 인스턴스 재활용
- **이벤트 처리**: 디바운싱으로 과도한 호출 방지
