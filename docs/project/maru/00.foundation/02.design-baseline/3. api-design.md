# MARU API 기본설계서

**프로젝트**: MARU (Master Code & Rule Management System)
**문서 버전**: 1.0
**작성일**: 2025-09-18
**문서 유형**: 기본설계서

---

## 1. 개요

### 1.1 목적
MARU 시스템의 RESTful API 설계 및 Frontend-Backend 간 인터페이스 규격을 정의한다.

### 1.2 범위
- API 설계 원칙
- 엔드포인트 정의
- 요청/응답 스키마
- 에러 처리 방식

---

## 2. API 설계 원칙

### 2.1 RESTful 설계
- HTTP 메서드별 명확한 의미 부여
- 리소스 중심의 URL 구조
- 무상태성 (Stateless) 보장
- 표준 HTTP 상태 코드 사용

### 2.2 URL 구조
```
/api/v1/{resource}[/{id}][/{sub-resource}][/{sub-id}]
```

### 2.3 HTTP 메서드 매핑
- **GET**: 조회 (안전한 오퍼레이션)
- **POST**: 생성
- **PUT**: 전체 수정 (멱등성)
- **PATCH**: 부분 수정
- **DELETE**: 삭제 (논리적 삭제)

## 3. 공통 응답 형식 (Nexacro Dataset XML)

### 3.1 성공 응답
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Dataset>
  <!-- 에러 정보 -->
  <ErrorCode>0</ErrorCode>
  <ErrorMsg></ErrorMsg>
  <SuccessRowCount>2</SuccessRowCount>

  <!-- 컬럼 정보 정의 -->
  <ColumnInfo>
    <Column id="COLUMN_NAME" type="STRING" size="100"/>
    <!-- 추가 컬럼 정의 -->
  </ColumnInfo>

  <!-- 실제 데이터 -->
  <Rows>
    <Row>
      <Col id="COLUMN_NAME">데이터 값</Col>
    </Row>
    <!-- 추가 행 데이터 -->
  </Rows>
</Dataset>
```

### 3.2 에러 응답
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Dataset>
  <ErrorCode>-400</ErrorCode>
  <ErrorMsg>입력값이 올바르지 않습니다.</ErrorMsg>
  <SuccessRowCount>0</SuccessRowCount>

  <ColumnInfo>
    <Column id="ERROR_FIELD" type="STRING" size="50"/>
    <Column id="ERROR_VALUE" type="STRING" size="100"/>
  </ColumnInfo>

  <Rows>
    <Row>
      <Col id="ERROR_FIELD">maruName</Col>
      <Col id="ERROR_VALUE">마루명은 필수입니다.</Col>
    </Row>
  </Rows>
</Dataset>
```

### 3.3 Nexacro ErrorCode 정의
| ErrorCode | 의미 | 사용 시나리오 |
|-----------|------|---------------|
| 0 | 성공 | 정상 처리 완료 |
| -1 | 일반 에러 | 데이터 없음, 조회 결과 없음 |
| -100 | 비즈니스 로직 에러 | 중복 데이터, 유효성 검사 실패 |
| -200 | 시스템 에러 | DB 연결 실패, 서버 오류 |
| -300 | 권한 에러 | 접근 권한 없음 |
| -400 | 파라미터 에러 | 필수값 누락, 형식 오류 |

### 3.4 HTTP 상태 코드 (전송 레벨)
모든 Nexacro 응답은 HTTP 200 OK로 전송되며, 실제 오류는 ErrorCode로 판단

## 10. Node.js Express 구현 예제

### 10.1 Nexacro XML 응답 생성 헬퍼
```javascript
// XML 응답 생성 헬퍼 함수
function createNexacroResponse(data, columns, errorCode = 0, errorMsg = '') {
  const xmlHeader = '<?xml version="1.0" encoding="UTF-8"?>';

  const columnInfo = columns.map(col =>
    `    <Column id="${col.id}" type="${col.type}" size="${col.size}"/>`
  ).join('\n');

  const rows = data.map(row => {
    const cols = columns.map(col =>
      `      <Col id="${col.id}">${row[col.id] || ''}</Col>`
    ).join('\n');
    return `    <Row>\n${cols}\n    </Row>`;
  }).join('\n');

  return `${xmlHeader}
<Dataset>
  <ErrorCode>${errorCode}</ErrorCode>
  <ErrorMsg>${errorMsg}</ErrorMsg>
  <SuccessRowCount>${data.length}</SuccessRowCount>

  <ColumnInfo>
${columnInfo}
  </ColumnInfo>

  <Rows>
${rows}
  </Rows>
</Dataset>`;
}
```

### 10.2 Express 라우터 구현 예제
```javascript
app.get('/api/v1/maru-headers', (req, res) => {
  try {
    const data = [
      {
        MARU_ID: 'DEPT_CODE_001',
        MARU_NAME: '부서코드',
        MARU_TYPE: 'CODE',
        MARU_STATUS: 'INUSE',
        VERSION: 2
      },
      {
        MARU_ID: 'USER_AUTH_001',
        MARU_NAME: '사용자권한코드',
        MARU_TYPE: 'CODE',
        MARU_STATUS: 'INUSE',
        VERSION: 1
      }
    ];

    const columns = [
      { id: 'MARU_ID', type: 'STRING', size: '20' },
      { id: 'MARU_NAME', type: 'STRING', size: '100' },
      { id: 'MARU_TYPE', type: 'STRING', size: '10' },
      { id: 'MARU_STATUS', type: 'STRING', size: '10' },
      { id: 'VERSION', type: 'INT', size: '4' }
    ];

    const xmlResponse = createNexacroResponse(data, columns);

    res.set('Content-Type', 'text/xml; charset=utf-8');
    res.send(xmlResponse);

  } catch (error) {
    const errorXml = createNexacroResponse([], [], -200, '시스템 오류가 발생했습니다.');
    res.set('Content-Type', 'text/xml; charset=utf-8');
    res.status(200).send(errorXml); // Nexacro는 항상 HTTP 200으로 응답
  }
});
```
