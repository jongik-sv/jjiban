## 1. 개요

### 1.1 목적
MARU 시스템의 전체적인 아키텍처 구조와 각 구성요소 간의 관계를 정의하여 개발팀이 일관된 기준으로 시스템을 구축할 수 있도록 한다.

### 1.2 범위
- 시스템 전체 아키텍처 구조
- 계층별 역할 및 책임
- 구성요소 간 인터페이스
- 데이터 흐름 및 처리 방식

## 2. 시스템 아키텍처 개요

### 2.1 아키텍처 패턴
**3-Tier Architecture** 적용
- **Presentation Tier**: Nexacro N v24 Frontend
- **Business Logic Tier**: Node.js + Express Backend
- **Data Tier**: Oracle Database + Cache Layer

## 3. 계층별 상세 설계

### 3.1 Presentation Layer (Nexacro Frontend)

#### 3.1.1 구성요소
- **Form Layer**: 사용자 인터페이스 화면
- **Dataset Layer**: 클라이언트 측 데이터 관리
- **Component Layer**: 재사용 가능한 UI 컴포넌트

#### 3.1.2 주요 책임
- 사용자 입력 수집 및 검증
- 서버로부터 받은 데이터 표시
- 비즈니스 로직 호출을 위한 API 요청
- 사용자 경험 최적화

### 3.2 Business Logic Layer (Node.js Backend)

#### 3.2.1 구성요소
```
┌─────────────┐
│   Router    │ ← RESTful API 라우팅
└─────────────┘
       │
┌─────────────┐
│ Controller  │ ← HTTP 요청/응답 처리
└─────────────┘
       │
┌─────────────┐
│  Service    │ ← 비즈니스 로직 구현
└─────────────┘
       │
┌─────────────┐
│   Model     │ ← 데이터 액세스 레이어
└─────────────┘
```

### 3.3 Data Layer (Oracle Database)

#### 3.3.1 구성요소
- **Core Tables**: 마스터 데이터 및 룰 관리
- **History Tables**: 이력 관리 (선분 이력 모델)
- **Indexes**: 성능 최적화
- **Constraints**: 데이터 무결성 보장
