## 2. ID 체계 정의

### 2.1 프로그램 ID 명명 규칙

#### 2.1.1 Frontend (Nexacro) 프로그램
```
[업무영역][기능구분][순번]
```

**업무영역 코드**:
- `MR`: 마루 관리
- `CD`: 코드 관리
- `RL`: 룰 관리
- `CM`: 공통/시스템 관리

**기능구분 코드**:
- `01`: 기본 관리 (CRUD)
- `02`: 조회/현황
- `03`: 이력/추적
- `04`: 검증/테스트
- `05`: 설정/환경

**순번**: 00-99

#### 2.1.2 Backend API
```
/api/v1/[resource]/[action]
```

#### 2.1.3 Database 객체
```
TB_MR_[객체유형]_[업무명]
```

### 2.2 파일명 규칙

#### 2.2.1 Nexacro N v24 Form
```
frm[프로그램ID].xfdl
```

#### 2.2.2 Backend 파일
```
[resource].controller.js
[resource].service.js
[resource].model.js
[resource].route.js
```

## 6. 프로젝트 폴더 구조

### 6.1 Frontend 구조 (Nexacro N v24)
```
nexacro/
├── Base/               # 공통 베이스
│   ├── Form/          # 공통 폼
│   ├── Component/     # 재사용 컴포넌트
│   └── Library/       # 공통 라이브러리
├── MR/                # 마루 관리
│   ├── MR01/         # 마루 헤더 관리
│   ├── MR02/         # 마루 현황 조회
│   └── MR03/         # 마루 이력 조회
├── CD/                # 코드 관리
│   ├── CD01/         # 코드 카테고리 관리
│   ├── CD02/         # 코드 기본값 관리
│   └── CD03/         # 코드 검증 관리
├── RL/                # 룰 관리
│   ├── RL01/         # 룰 변수 관리
│   ├── RL02/         # 룰 레코드 관리
│   └── RL03/         # 룰 실행 테스트
└── CM/                # 공통 관리
    ├── CM01/         # 메인화면
    ├── CM02/         # 사용자 관리
    └── CM03/         # 시스템 설정
```

### 6.2 Backend 구조 (Node.js)
```
backend/
├── config/            # 설정 파일
├── routes/            # API 라우트
│   ├── maruHeader.js
│   ├── codeCategory.js
│   ├── codeBase.js
│   ├── ruleVariable.js
│   └── ruleRecord.js
├── controllers/       # 컨트롤러
│   ├── maruHeader.controller.js
│   ├── codeCategory.controller.js
│   ├── codeBase.controller.js
│   ├── ruleVariable.controller.js
│   └── ruleRecord.controller.js
├── services/          # 서비스 레이어
│   ├── maruHeader.service.js
│   ├── codeCategory.service.js
│   ├── codeBase.service.js
│   ├── ruleVariable.service.js
│   └── ruleRecord.service.js
├── models/            # 데이터 모델
│   ├── maruHeader.model.js
│   ├── codeCategory.model.js
│   ├── codeBase.model.js
│   ├── ruleVariable.model.js
│   └── ruleRecord.model.js
├── middleware/        # 미들웨어
├── utils/             # 유틸리티
├── migrations/        # DB 마이그레이션
└── seeds/             # 초기 데이터
```

## 7. 개발 우선순위

### 7.1 Phase 1 (핵심 기능)
1. **MR0100** - 마루헤더관리 (Backend: MH001-MH006)
2. **CD0100** - 코드카테고리관리 (Backend: CC001-CC005)
3. **CD0200** - 코드기본값관리 (Backend: CB001-CB005)

### 7.2 Phase 2 (룰 관리)
4. **RL0100** - 룰변수관리 (Backend: RV001-RV005)
5. **RL0200** - 룰레코드관리 (Backend: RR001-RR005)
6. **RL0300** - 룰실행테스트 (Backend: RR006)

### 7.3 Phase 3 (조회 및 관리)
7. **MR0200** - 마루현황조회
8. **MR0300** - 마루이력조회 (Backend: MH007)
9. **CD0300** - 코드검증관리 (Backend: CB006)

### 7.4 Phase 4 (시스템 관리)
10. **CM0100** - 메인화면
11. **CM0200** - 사용자관리 (향후)
12. **CM0300** - 시스템설정 (향후)
