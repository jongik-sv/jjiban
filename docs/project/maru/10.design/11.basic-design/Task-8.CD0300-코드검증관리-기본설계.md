## 1. 무엇을 만드나?

**코드 검증 화면**: 코드 카테고리 정의와 실제 코드값이 일치하는지 자동으로 체크해주는 기능

---

## 2. 주요 기능

### 2.1 기능 목록

1. **검증 실행**: 마루 선택하고 "검증" 버튼 누르면 검사 시작
2. **결과 표시**: 문제 있는 코드만 빨간색으로 표시
3. **상세 보기**: 문제 있는 코드 클릭하면 뭐가 잘못됐는지 팝업으로 보여줌
4. **Excel 저장**: 결과를 Excel로 내보내기

### 2.2 검증 규칙

**Normal 타입 (콤마 구분)**:
```
카테고리 정의: "1,2,3,4,5"
→ 코드값은 1, 2, 3, 4, 5 중 하나여야 함
→ "LEVEL99" 같은 건 에러
```

**RegEx 타입 (정규식)**:
```
카테고리 정의: "^[A-Z]{2}[0-9]{4}$"
→ 코드값은 "AB1234" 형식이어야 함
→ "dept001" 같은 건 에러
```

**중복 검사**:
```
→ 같은 코드가 2개 이상 있으면 에러
```

## 3. 화면 설계

### 3.1 화면 레이아웃

```
┌─────────────────────────────────────────┐
│ 마루 선택: [드롭다운▼]  [검증 실행]     │
├─────────────────────────────────────────┤
│ 전체 150건 | 정상 140건 | 오류 10건    │
├─────────────────────────────────────────┤
│ 필터: ○ 전체  ● 오류만                  │
├─────────────────────────────────────────┤
│ 카테고리 | 코드  | 코드명 | 상태 | 상세 │
│ ─────────────────────────────────────── │
│ 부서레벨 | 1     | 1급   | ✓   |      │
│ 부서레벨 | LVL99 | 최고급 | ✗   | [보기] │
│ 부서코드 | dept1 | 경영팀 | ✗   | [보기] │
└─────────────────────────────────────────┘
```

### 3.2 오류 상세 팝업

```
┌─────────────────────────────┐
│ 뭐가 문제인가요?             │
├─────────────────────────────┤
│ 코드: LVL99                 │
│ 카테고리: 부서레벨 (Normal)  │
│                             │
│ 문제:                       │
│ 허용값이 아닙니다.          │
│ 허용값: 1,2,3,4,5           │
│                             │
│ [닫기]                      │
└─────────────────────────────┘
```

## 4. 데이터 주고받기

### 4.1 Backend에 요청

```
POST /api/v1/maru-headers/{마루ID}/validate-codes

→ 검증 시작
```

### 4.2 Backend에서 받는 응답


## 5. 어떤 경우에 에러가 나나?

| 상황 | 에러 메시지 |
|------|-------------|
| 마루를 안 골랐을 때 | "마루를 선택해주세요" |
| 서버 연결 안 될 때 | "서버 연결 실패. 다시 시도하세요" |
| 코드가 너무 많을 때 (1000개 이상) | "시간이 오래 걸릴 수 있습니다. 계속하시겠습니까?" |
| 정규식이 잘못됐을 때 | "코드 카테고리 정의가 잘못되었습니다" |

## 6. 성능 목표

- 100개 코드 검증: **1초 이내**
- 1000개 코드 검증: **5초 이내**
- 화면 표시: **바로 보임** (1초 이내)
